# 114-4 北北基學測國文模擬考試

> **內容管理**：所有網頁文字（題目、選項、解析、單元說明、入口頁卡片）均集中在 `content.md`。修改它，其他都不用碰。

---

## ✏️ 修改考試內容

**唯一要編輯的檔案：**

```
web/114-4-chinese/content.md
```

---

## 🚀 本地開發（改 MD → 瀏覽器自動刷新）

```powershell
cd web/114-4-chinese
npm run dev
```

瀏覽器開啟 **http://localhost:3456/**，存檔 `content.md` 後頁面自動刷新。

---

## 🔨 Build（更新靜態 JS 資料）

```powershell
cd web/114-4-chinese
npm run build
```

這會重新產生 `course-data.js`（考試頁資料）和 `../portal-data.js`（入口頁資料）。

---

## 🌐 部署到 GitHub Pages

每次修改 `content.md` 後執行：

```powershell
cd web/114-4-chinese
npm run build

# 在 web/ 的 git repo 下 push
cd ..
git add -A
git commit -m "content: update"
git push
```

`course-data.js` 和 `portal-data.js` 都是靜態 JS，直接由 GitHub Pages 提供，無需任何 server-side 功能。

## 📝 content.md Markdown 語法與 HTML 網頁呈現對照指南

`content.md` 是本專案唯一的內容來源，其內容透過 `md-to-data.js` 解析，並由前端 `index.html` 渲染。以下整理其 Markdown 語法格式與 HTML 呈現規則的對照表：

### 1. 全局配置區段 (`# META`)

此區段定義整個考場的基礎 metadata 與入口說明資訊。

| Markdown 語法 / YAML 欄位 | HTML 網頁呈現位置與功能 | 說明 / 注意事項 |
| :--- | :--- | :--- |
| **YAML** `title: ` | 1. 考試介紹頁大標題 `<h1>`<br>2. 瀏覽器網頁分頁 `<title>` | 考場主名稱 |
| **YAML** `subtitle: ` | 考試介紹頁副標題（大標題底下的前導文字） | 用以說明考卷類型或簡介 |
| **YAML** `program: ` | 側邊欄最上方的分類標籤（如「學術能力模擬檢測」） | 用於考科或系列檢測的分類標籤 |
| **YAML** `organizer: ` | 考試簡介資訊表格中的「主辦單位」欄位 | 顯示主辦或出題單位名稱 |
| **YAML** `dates: ` | 考試簡介資訊表格中的「建議時程」欄位 | 顯示考卷適用學年度或日期 |
| **YAML** `format: ` | 考試簡介資訊表格中的「作答說明」欄位第一行 | 顯示測驗形式 |
| **YAML** `instructions: ` | 考試簡介資訊表格中「作答說明」的詳細描述段落 | 若不設定，系統會採用預設的國綜考場說明 |
| **YAML** `storeKey: ` | 瀏覽器 `localStorage` 的 Key 值 | **重要**：每份試卷必須唯一。系統以此 key 儲存作答進度、對錯與即時批改狀態，若與其他試卷重複會導致進度覆蓋！ |
| **YAML** `timerSeconds: ` | 右上角倒數計時器的秒數（例如 `5400` 秒 = 90 分鐘） | 控制倒數計時。時間到時會強制執行交卷並結算總分 |
| **Markdown 列表** `## 學習目標` | 考試簡介頁面的「線上測驗學習目標」無序清單 | 以 `-` 開頭的 Bullet Points 呈現 |
| **Markdown 表格** `## 考科時程` | 考試簡介頁面的「考科時程與配置」表格 | 建議時間/單元/重點對照，支援 Markdown 表格語法 |

### 2. 考試單元與導航區段 (`# DAY1`)

此區段主要定義側邊欄的導覽章節、各單元（風琴摺 Accordion）的介紹內容及學習清單。

| Markdown 語法 / YAML 欄位 | HTML 網頁呈現位置與功能 | 說明 / 注意事項 |
| :--- | :--- | :--- |
| **YAML** `title: ` | 1. 單元頂部大 Banner (Hero) 的標題<br>2. 側邊欄的單元連結名稱 | 點選側邊欄會平滑滾動 (Smooth Scroll) 至此單元 |
| **YAML** `learningGoal: ` | 單元大 Banner (Hero) 底下的前導說明文字 | 說明本單元的作答與學習重點 |
| **Heading 2** `## 單元一：[標題]` | 題目作答區中的**風琴摺 (Accordion) 標頭** | 點擊標頭可展開/收合該單元的考題與引導內容 |
| **YAML** `id: ` | 生成 DOM 的 `id` 與側邊欄對應的 progress pill 綁定 | 第一單元設為 `u-1`，第二單元為 `u-2`，第三單元為 `u-3`。進度小膠囊（如 `0 / 24`）會與此 ID 的題目作答狀態綁定 |
| **YAML** `subtitle: ` | 風琴摺標頭內的說明文字 | 標示此單元包含的題號範圍與配分（例如：第 1 題至第 24 題，每題 2 分） |
| **YAML** `time: ` | 風琴摺標頭內的建議用時 | 提供時間分配參考 |
| **Heading 3** `### 任務清單` | 展開風琴摺後顯示的互動任務清單 | 使用 `- [d1-u1-t1] 任務名稱`。在網頁上會渲染為**可點選的 Checkbox**，點擊後會即時儲存勾選狀態並影響側邊欄完成度！ |
| **Heading 3** `### 素材` | 展開風琴摺後顯示的相關素材下載連結 | 使用 `- [MD] 檔案名 \| 說明`。網頁上會渲染為**小膠囊連結**，點選會彈出 iframe 試卷閱讀器直接在網頁上查看 |
| **Heading 3** `### 插圖` | 展開風琴摺後顯示的示意圖插畫 | 使用 `- [圖片ID] \| hero \| 說明`。網頁中會自動依據 `圖片ID` 渲染對應的動態設計區塊（或 SVG 示意圖） |

### 3. 試卷題目區段 (`# QUIZ`)

核心答題區，定義所有考題、選項、正確答案與即時解析。題目格式共有三種，由 Heading 2 標題文字與 YAML 中的欄位自動偵測：

#### 💡 (1) 單選題 (Single Choice)

*   **偵測規則**：Heading 2 包含 `單選題` 或 `混合單選`，且 YAML 中**無** `type: text` 欄位。
*   **Markdown 語法範例**：
    ```markdown
    ## Q1 · 單選題
    下列「 」內的字，讀音前後相同的是：
    - (A)「裨」補闕漏／彈箏搏「髀」
    - (B)目「眥」盡裂╱所費不「貲」
    - (C)人為刀「俎」／「狙」公賦芧
    - (D)橫「槊」賦詩／不「溯」既往

    ```yaml
    answer: A
    unit: day1.u-1
    explanation: |
      【答案】(A)
      【解析】...
    ```
    ```
*   **網頁呈現與互動規則**：
    1.  **題幹與選項**：渲染為題目字型與 `(A)` ~ `(D)` 的選項按鈕。
    2.  **答題即時回饋**：
        *   點選任一選項後，該題**立即鎖定**，不可再更改答案。
        *   **對錯判定**：如果答對，該題外框與所選選項變為**綠色（含 ✓ 符號）**；若答錯，所選選項變為**紅色**，且正確選項會以**綠色 ✓** 標明。
        *   **解析呈現**：底下的 `explanation` 區塊（解析盒）會**立即展開**，且其左側框線會根據答對/答錯呈現綠色/紅色。
    3.  **進度同步**：側邊欄進度條與單元小膠囊（例如 `pill-u-1`）會立即更新進度。

#### 💡 (2) 多選題 (Multiple Choice)

*   **偵測規則**：Heading 2 包含 `多選題`，且 YAML 中**無** `type: text` 欄位。
*   **Markdown 語法範例**：
    ```markdown
    ## Q25 · 多選題
    下列各組「 」內的詞，意義前後相同的是：（多選題，全對得 4 分）
    - (A)何故深思高「舉」...
    - (B)便扶「向」路...
    - (C)五十絃翻塞外聲...
    - (D)生乎吾前...
    - (E)軒凡四遭火...

    ```yaml
    answer: [B, D, E]
    unit: day1.u-2
    explanation: |
      【答案】(B)(D)(E)
      【解析】...
    ```
    ```
*   **網頁呈現與互動規則**：
    1.  **複選機制**：允許使用者自由點選 `(A)` ~ `(E)` 多個選項，點擊時會呈現藍色選取狀態。
    2.  **確認與即時回饋**：
        *   題目下方會渲染一個「**確認選項並看解析**」的按鈕。
        *   點擊確認後，該題鎖定，按鈕消失，並**立即顯示對錯與解析**。
        *   **對錯判定**：依據大考中心多選計分公式判定（全對得 4 分，錯 1 個選項得 2.4 分，錯 2 個得 0.8 分，錯 3 個以上 0 分）。全對外框為綠色，有錯則為紅色。所有正確的選項皆會標示為綠色 ✓。
        *   **解析呈現**：`explanation` 解析盒立即滑出展開。

#### 💡 (3) 手寫非選題 / 混合簡答題 (Text / Free Response)

*   **偵測規則**：Heading 2 包含 `手寫`、`非選擇`、`表格`、`簡答`、`混合題` 且 YAML 區段包含 `type: text` 與 `subQuestions`。
*   **Markdown 語法範例**：
    ```markdown
    ## Q34 · 手寫混合題
    【手寫混合題】二文皆藉特定景物寄託對亡妻的思念。請指出各自所用的景物及其象徵意涵。

    ```yaml
    type: text
    unit: day1.u-3
    subQuestions:
      - label: "① 〈項脊軒志〉所用景物 (5字以內)"
        placeholder: 請輸入所用景物
      - label: "② 〈項脊軒志〉該景物的象徵意涵 (5字以內)"
        placeholder: 請輸入象徵意涵
    answer:
      "0": 枇杷樹
      "1": 承載夫妻回憶 / 象徵生命延續
    explanation: |
      【評分標準】...
    ```
    ```
*   **網頁呈現與互動規則**：
    1.  **手寫輸入框**：依據 `subQuestions` 中的 `label` 渲染出對應個數的**文字輸入框 (Textarea)**。
    2.  **字數監控**：輸入框右下角會**即時計算字數**，字數超過限制時可輔助提醒。
    3.  **自主對照答案**：
        *   此類題型不進行自動對錯批改。下方會渲染「**顯示參考答案與評分規準**」按鈕。
        *   點擊按鈕後展開 `explanation` （包含參考解答與詳細的評分標準），供學生自行比對。

### 4. 入口頁配置區段 (`# PORTAL`)

此區段在 `web/portal.md` 中獨立維護，用以控制考試平台入口儀表板（`web/index.html`）的呈現。

*   **YAML 區段**：定義入口頁大標題與歡迎文字。
*   **Heading 2 `## Card: [資料夾名稱]`**：
    *   `href` 指向對應的考場資料夾路徑。
    *   YAML 包含該試卷在入口儀表板卡片上顯示的 `tag`（科目標籤）、`title`（卡片標題）、`desc`（考試簡介）、`time`（作答時間）、`questions`（總題數）、`btnText`（按鈕文字）。

## ➕ 如何新增試卷

1. 在 `web/` 下建立新資料夾，例如 `web/114-4-english/`。
2. 將 `114-4-chinese/` 的下列檔案複製過去：
   `index.html`、`style.css`、`viewer.html`、`package.json`、`md-to-data.js`、`dev-server.js`
3. 建立新的 `content.md`，填寫 `# META`、`# DAY1`、`# QUIZ` 區段。
4. 執行 `npm run build` 產生試卷資料。
5. 在 `web/portal.md` 新增一張考試卡片，再執行 `node scripts/build-portal.js` 重新生成入口儀表板卡片。
